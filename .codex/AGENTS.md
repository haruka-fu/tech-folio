# リポジトリ運用ガイドライン

このドキュメントは、`tech-folio` に貢献する開発者および AI エージェント向けのガイドです。

## プロジェクト構成とモジュール

- アプリケーションのルーティングと UI は `app/`（Next.js App Router）にあります。
- 共通ロジックや型定義は `lib/` に配置します。
- 画像やフォントなどの静的アセットは `public/` 配下に置きます。
- 仕様書や設計書などのドキュメントは `docs/` にまとめます。
- `.next/` の生成物や `node_modules/` の依存パッケージは直接編集しないでください。

## ビルド・テスト・開発

- `npm run dev`  Eローカル開発サーバーを `http://localhost:3000` で起動。
- `npm run build`  E本番用ビルド（デプロイ前に実行）。
- `npm start`  Eビルド済みアプリをローカルで起動。
- `npm run lint`  EESLint を実行し、指摘はコミット前に解消してください。
- Docker: 本番イメージは `docker build -t tech-folio .`、開発用ホットリロード環境は `docker compose -f docker-compose.dev.yml up` を使用します。

## コーディングスタイルと命名

- TypeScript・React 関数コンポーネント・フックを基本とします。
- 既存コードに合わせてインデントは 2 スペース、文字列・インポートにはダブルクォートを優先します。
- コンポーネントは関心ごとを絞り、関連するスタイルやロジックは `app/` 配下の同一フォルダにまとめてください。
- 変数・関数名はキャメルケース、コンポーネントと React ファイル名はパスカルケースでわかりやすく命名します。

## テスト方針

- 現時点では、このリポジトリに公式な自動テストスイートはありません。
- テストを追加する場合は、対象機能の近く（例: `app/feature/__tests__/…`）に配置し、`npm test` スクリプトから実行できるようにしてください。
- PR を出す前に、最低限 `npm run dev` で主要な画面とフローを手動確認してください。

## コミットとプルリクエスト

- コミットメッセージは短く内容が分かる日本語または英語で書きます（例: `プロフィールカードのレイアウトを調整`, `Add skill aggregation summary to dashboard`）。
- 関連する変更は 1 つのコミットにまとめ、リファクタリングと機能追加を混在させないようにしてください。
- プルリクエストでは変更内容・目的・画面への影響を簡潔に説明し、関連 Issue があれば `Closes #123` のようにリンクします。UI 変更時は可能であればスクリーンショットを添付してください。
